# 05 - Node.js Architecture

## About

This project demonstrates the core architecture of Node.js and how it efficiently handles multiple client requests. It explores the request lifecycle, the Event Queue, Event Loop, Thread Pool, and the difference between blocking and non-blocking operations.

## What I Learned

- How Node.js processes client requests
- Role of the Event Queue and Event Loop
- Blocking vs Non-Blocking operations
- How the Thread Pool handles time-consuming tasks
- Default Thread Pool size and its impact on scalability
- Why asynchronous programming is preferred in Node.js
- Using the built-in `os` module to check the number of CPU cores

## Files

- `architecture.js` – Displays the number of logical CPU cores using the `os` module.
