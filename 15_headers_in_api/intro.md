# HTTP Headers & APIs

## What are HTTP Headers?

HTTP Headers are additional pieces of information exchanged between the client and the server.

Think of them as the **metadata** of an HTTP request or response.

They do **not** contain the actual data (body). Instead, they describe information about the data being transferred.

Examples include:

```http
Content-Type: application/json
User-Agent: Mozilla/5.0
Authorization: Bearer xxxxx
Accept: */*
```

---

## Real Life Analogy

Imagine **Friend 1** wants to send a letter to **Friend 2**.

The actual message is written **inside** the envelope.

The envelope itself contains additional information such as:

- From
- To
- Address
- Stamp
- Priority

The receiver first reads the information on the envelope before opening the letter.

HTTP works in a similar way.

The **Body** contains the actual data, while the **Headers** contain extra information that helps the client and server understand how the data should be handled.

```
Envelope
-------------------------
From:
To:
Priority:
Stamp:
-------------------------

Message inside the envelope
```

# Request and Response

Whenever a browser, Postman, or any application communicates with a server, two things happen.

```
Client
   │
   │ Request
   ▼
Server
   │
   │ Response
   ▼
Client
```

Both the request and the response contain their own headers.

---

# Request Headers

Request headers travel from the **Client → Server**.

They tell the server information such as:

- Browser details
- Content Type
- Authentication
- Cookies
- Language
- Cache information
- Custom headers

Example

```http
Content-Type: application/json
User-Agent: PostmanRuntime/7.53.0
Authorization: Bearer Token
Accept: */*
```

---

# Response Headers

Response headers travel from the **Server → Client**.

They describe the response generated by the server.

Example

```http
Content-Type: application/json
Content-Length: 420
Date: Tue, 08 Jul 2026
```

Servers can also send custom headers.

```javascript
res.setHeader("X-MyName", "Shalini Kumari");
```

---

# APIs

API stands for **Application Programming Interface**.

An API allows two software applications to communicate with each other.

Examples:

```
GET /users
POST /users
PATCH /users/1
DELETE /users/1
```

A client sends a request, and the server responds with data.

---

# Request Headers in Postman

Open **Postman**

Go to the **Headers** tab.

You can create your own custom headers.

Example

| Key | Value |
|------|-------|
| Purpose | Learning Backend |

Now inside Express,

```javascript
console.log(req.headers);
```

Output

```javascript
{
  purpose: 'Learning Backend',
  'user-agent': 'PostmanRuntime/7.53.0',
  accept: '*/*',
  host: 'localhost:8000',
  ...
}
```

---

# Response Headers in Postman

When the server sends a response,

Go to

```
Response
    ↓
Headers
```

You'll see both the standard headers and any custom headers sent using

```javascript
res.setHeader()
```

For example

```
X-MyName: Shalini Kumari
```

# Browser Developer Tools

Headers are not only visible in Postman.

You can inspect them in any browser.

### Google Chrome / Microsoft Edge

```
Right Click
      ↓
Inspect
      ↓
Network
      ↓
Select any request
```

You'll find

- Request Headers
- Response Headers

Even websites like **YouTube** use hundreds of headers while communicating with their servers.

---

# Why are Headers Important?

Headers help in

- Authentication
- Sending Content Type
- Cookies
- Caching
- Compression
- Security
- Language preferences
- Custom application data

Without headers, the client and server wouldn't know how to process the incoming data correctly.
